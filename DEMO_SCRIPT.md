# 演示脚本 — AI简历优化助手

**演示时长目标：90秒内完成**

---

## 演示前准备（30秒前完成）

```bash
# 终端1：启动后端
cd D:/AI/BS0326/backend && node server.js

# 终端2：启动前端
cd D:/AI/BS0326/frontend && npm run dev
```

打开浏览器，访问：**http://localhost:5173**

---

## 演示步骤

### 第1步：首页亮点展示（15秒）

**打开首页（Landing Page）**

口播要点：
> "这是AI简历优化助手。求职者每次投简历都在猜，我们把这个过程量化——你的简历和这个JD到底匹配多少分？哪些关键词缺失？怎么改？AI全部帮你搞定。"

- 指向三个功能卡片：**匹配分析 / 关键词提取 / AI重写**
- 点击「开始分析」进入输入页

---

### 第2步：一键演示（10秒）

**在输入页（Input Page）**

口播要点：
> "这里粘贴简历和目标职位JD。我们现在用一键演示功能，用一个真实的求职场景来展示。"

- **点击「一键演示」按钮**（自动填入张伟的简历 + 字节跳动JD）
- 快速浏览两个文本框内容，无需逐字读
- **点击「开始分析」**，进入AI分析流程

---

### 第3步：分析过程（10秒）

**Loading动画展示**

口播要点：
> "AI正在同步做三件事：提取关键词、多维度打分、生成优化建议和改写简历。整个过程大约10秒。"

- 指向进度消息的切换动画

---

### 第4步：结果展示（40秒）

**匹配得分（Score Tab）**
> "72分——基本匹配。技能匹配68分，这里是拉分项，因为缺TypeScript和Redux。学历匹配85分，这是加分项。"

**关键词分析（Keywords Tab）**
> "你看，绿色是已匹配的10个关键词，红色是缺失的——TypeScript、Redux、CI/CD——这些正是JD里明确要求的。"

**优化建议（Suggestions Tab）**
> "AI给出了6条建议，按高中低优先级排序。最高优先：补充TypeScript经验；其次：给业绩加数据量化。具体到哪个模块，怎么改，一目了然。"

**AI重写简历（Rewrite Tab）**
> "这是AI基于JD直接重写的简历版本。注意这里——原来模糊的'优化性能'变成了'FCP从3.2s降至1.5s，提升53%'。点复制，直接用。"

---

### 第5步：收尾（15秒）

口播要点：
> "整个流程不到2分钟。我们后续计划接入多JD批量分析，以及语音面试复盘功能。这是AI真正帮到求职者的第一步——不再盲投，精准优化。"

- 返回首页，展示产品整体视觉

## 备用方案（若AI接口超时）

在浏览器控制台执行：
```js
// 切换到演示模式，直接使用预设数据
localStorage.setItem('forceDemo', 'true')
location.reload()
```

或重启后端并设置演示模式：
```bash
DEMO_MODE=true node server.js
```
